HTML Semantics Cheatsheet

Semantics in HTML gives meaning to content, improves accessibility and SEO, and makes the code clearer and more organized.

Semantic Meaning and usage
<header> The semantic meaning of the header tag is that it represents the introductory section or header of a page or section.
<nav> Nav semantic indicates that the content contains main navigation links within a page or website.
<main> It is used to identify the main and unique content of the webpage, which is directly related to the document's topic.
<footer> Contains information related to closing a page or section, or additional information about it.
<section> It is used to divide the content into related thematic sections within a page.
<article> Refers to standalone content that can exist on its own, such as an article, news story, or post.
<aside> It is used for secondary or supplementary content related to the main content.
<figure> It is used to group visual or illustrative content related to the text, usually accompanied by a description using the <figcaption> tag.
<figcaption> This is used to add a caption or description to the media encapsulated in the aforementioned tag.
<video> Is the HTML tag that allows you to embed and play videos on a web page.
<embed> It is used to embed external content within a web page, such as PDFs or multimedia files.
<audio> It is used to play audio files directly on the webpage, with controls and support for multiple formats.

This is not a complete list of all existing semantics. You can find more at MDN Semantics Glossary
